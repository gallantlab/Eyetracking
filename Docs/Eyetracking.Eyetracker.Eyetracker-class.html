WriteVideoWithGazePosition(self, frames, fileName, dataStart=None, firstFrame=None, outResolution=(1024, 768), flipColors=True, fps=30)

source code 

Writes out a video using the input frames and given eyetracking. If neither dataStart nor firstFrame are provided, then the assumption is that eyetracking calibration is included in the data, and that the data start is the eyetracking calibration start.

Parameters:
  • frames ([t x h x w x 3] numpy.ndarray) - assumed to be same fps as the eyetracking
  • fileName (str) - name of output file
  • dataStart (tuple<int, int, int, int>) - timestamp of the start of the data frames, has precedence over firstFrame
  • firstFrame (int?) - the frame number in the eyetracking that the first frame of the video corresponds to
  • outResolution (tuple<int, int>) - desired output resolution
  • flipColors (bool) - flip frame color channels order? original videos are in BGR
  • fps (float) - float, fps of the frames

Writes out a video in which the frames are moved such that the gaze position is always the center

Parameters:
  • frames ([t x w x h x 3] array) - video frames
  • fileName (str) - name of output file
  • scale (float) - scale of these frames relative to the eyetracking scale (typically 1024 x 768)
  • dataStart (tuple<int, int, int, int>) - timestamp of the start of the data frames, has precedence over firstFrame
  • firstFrame (int?) - the frame number in the eyetracking that the first frame of the video corresponds to
  • flipColors (bool) - flip frame color channel order?
  • padValue (float) - range [0, 1] value to use for padding in the recentered frames
  • gazeSize (tuple<int, int>) - stimuli resolution on to which eyetracking is mapepd
  • fps (float) - fps of the frames
